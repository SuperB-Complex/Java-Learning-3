ThreadLocal Example with Java 8

Posted on May 29, 2017 Updated on January 3, 2020

What is ThreadLocal in Java?

ThreadLocal is a specially provisioned functionality by JVM to provide isolated storage space for threads only. like the value of instance scoped variable are bound to a given instance of a class only. each object has its only values and they can not see each other value. so is the concept of ThreadLocal variables, they are local to the thread in the sense of object instances other thread except for the one which created it, can not see it.

Like request, session and application scope of any Object, We can define a Thread Scope of any Object thus Object visibility would be only within the Thread and Every Thread has it execution life cycle, After a thread goes away, all of its copies of thread-local instances are subject to garbage collection (unless other references to these copies exist).

Although we usually create a Synchronize  Object for Thread safety in MultiThread application therefore only one Thread can be obtained a monitor to access shared Object Since shared Object is not thread-safe.

In Java 8, A new method is introdued  to determine the initial value.

public static  ThreadLocal withInitial(Supplier<? extends S> supplier) {
    return new SuppliedThreadLocal<>(supplier);
}

Here, a simple ThreadLocal example of Java 8 is demonstrated that how each Thread gets own ThreadLocal copy of variable and value remains the same even in the Iteration process and each thread is executing separately on the ThreadLocal variable.

import java.util.concurrent.atomic.AtomicInteger;
import java.util.stream.IntStream;

/**
 * Created by MYPC on 5/29/2017.
 */
public class ThreadLocalExample {
    private static final AtomicInteger transactionId = new AtomicInteger(10000);
    private static final ThreadLocal threadLocal = ThreadLocal.withInitial(() -> transactionId.getAndIncrement());

    public static void main(String[] args) {

        // Thread 1 executing and getting same Transaction Id for all iteration
        new Thread(() -> IntStream.range(0, 3).forEach(val -> {
            System.out.println(Thread.currentThread().getName() + " > " + threadLocal.get());
        })).start();

        // Thread 2 executing and getting same Transaction Id for all iteration
        new Thread(() -> IntStream.range(0, 3).forEach(val -> {
            System.out.println(Thread.currentThread().getName() + " > " + threadLocal.get());
        })).start();
        // Thread 3 executing and getting same Transaction Id for all iteration
        new Thread(() -> IntStream.range(0, 3).forEach(val -> {
            System.out.println(Thread.currentThread().getName() + " > " + threadLocal.get());
        })).start();

    }
}

Here are some snippet codes of ThreadLocal that could be used in the Advance level. Here last Thread calls use ThreadLocal and share the value to all methods inside Thread i.e HttpRequest call in which keep/generate transaction Id or Authentication Token in ThreadLocal and spare into all subsequent method calls inside the Thread.

package com.example.current;

import java.io.IOException;
import java.util.UUID;
import java.util.concurrent.atomic.AtomicInteger;


 public class ThreadLocalTest{
     public static void main(String[] args) {
         AuthTokenContextTest tokenContextTest = new AuthTokenContextTest();
         new Thread(()->tokenContextTest.execute(Thread.currentThread().getName())).start();
         new Thread(()->tokenContextTest.execute(Thread.currentThread().getName())).start();
         new Thread(()->tokenContextTest.execute(Thread.currentThread().getName())).start();
         new Thread(()->tokenContextTest.execute(Thread.currentThread().getName())).start();
         new Thread(()->tokenContextTest.execute(Thread.currentThread().getName())).start();
         new Thread(()->tokenContextTest.execute(Thread.currentThread().getName())).start();

         new Thread(()->{

             tokenContextTest.execute(Thread.currentThread().getName());
             tokenContextTest.execute(Thread.currentThread().getName());
             tokenContextTest.execute(Thread.currentThread().getName());
             tokenContextTest.execute(Thread.currentThread().getName());
             tokenContextTest.execute(Thread.currentThread().getName());

         }).start();


     }
 }

 class AuthTokenContextTest {

    private final AuthTokenContext authTokenContext = new AuthTokenContext(new LoginAuthTokenService());


    public void execute(String message)  {
        String authToken = authTokenContext.get();
        System.out.println(" Message is " + message + ", token is " + authToken);
    }

    static class AuthTokenContext {
        private static ThreadLocal THREAD_CONTEXT;

        public AuthTokenContext(LoginAuthTokenService loginAuthTokenService) {
            THREAD_CONTEXT = ThreadLocal.withInitial(() -> loginAuthTokenService.getAuthToken());
        }

        public void set(LoginAuthTokenService loginAuthTokenService) {
            THREAD_CONTEXT.set(loginAuthTokenService.getAuthToken());
        }

        public String get() {
            return THREAD_CONTEXT.get();
        }

    }

    final class LoginAuthTokenService {
        private String login() throws IOException {
            return UUID.randomUUID().toString();
        }

        public String getAuthToken() {
            AtomicInteger retryCount = new AtomicInteger(1);
            while (true) {
                try {
                    synchronized (LoginAuthTokenService.class) {
                        return login();
                    }
                } catch (Exception e) {
                    System.out.println(e);
                }
            }
        }
    }
}
