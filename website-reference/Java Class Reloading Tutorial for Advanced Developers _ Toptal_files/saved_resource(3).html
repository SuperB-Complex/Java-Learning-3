                        A nice addition to the article would talk about how JRebel relates and maybe the steps to creating your own version of JRebel? Another good addition would be to discuss how the JVM does (or doesn't) update all the references to a class and its instances in memory after re-loading a class. Lastly, instead of using 2 second loop, I recommend using a TCP client in your program to listen for file change events, and then you can send messages from a dev TCP server to whatever JVM is running on your machine. Basically you have a non-polling file-watcher process, and it sends a TCP message to any JVM which is listening for messages.
